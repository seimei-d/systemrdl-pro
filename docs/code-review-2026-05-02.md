## Cross-cutting themes (≥3 agents independently flagged)

1. **`expandNode` race conditions** (3 reviews): keying on `nodeId` without version, resolver overwrite on rapid clicks, no panel-disposal cancellation.
2. **Module-level state shared across panels** (2 reviews): `cursorSyncTimer`, `suppressNextCursorSync`, `flashDecoration` global singletons → multi-panel drops events on the wrong panel.
3. **Test hooks bleeding into production** (2 reviews): `_systemrdl_state` / `_systemrdl_full_pass_async` attached to live `server` object; `__all__` 70 lines as test-import registry.
4. **Disposal hygiene** (2 reviews): `FileSystemWatcher`, `TextEditorDecorationType`, panel listeners registered in wrong scope → leak on restart / mismatched lifetimes.
5. **Inline transport JS not type-checked** (3 reviews — extension, CLI, viewer-core): the JS that bridges webview/iframe to host is a string literal; drifts silently from viewer-core's `Transport` interface.
6. **Defensive `.catch(() => {})` swallows real errors** (2 reviews): viewer-core `getTree`, several extension paths.
