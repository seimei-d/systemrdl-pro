# SystemRDL examples

Each `.rdl` file here exercises one or more LSP/viewer features. Open any
of them in VSCode (`Ctrl+Shift+V` ‚Üí Memory Map) and try the actions
described inside the file's header comment.

| File | What it covers |
|---|---|
| [`sample.rdl`](sample.rdl) | Multi-feature SystemRDL demo ‚Äî two top-level addrmaps (`cpu_top`, `dma_engine`, `uart`), nested addrmap/regfile containers, named field types, reused `dma_channel_t` regfile. Default starting point. |
| [`features_demo.rdl`](features_demo.rdl) | Comprehensive showcase: user-defined property, enums + `encode` binding, signals, parametrized type, counter / interrupt fields, default-property propagation, alias register, `bridge` addrmap, `ispresent` conditional, `accesswidth` split-access, dynamic property assignment. |
| [`enum_demo.rdl`](enum_demo.rdl) | Minimal `enum` + `encode = ‚Ä¶` field binding. Detail panel shows the enum value table inside the field row. |
| [`alias_demo.rdl`](alias_demo.rdl) | `alias <orig> <type> <new> @ <addr>;` ‚Äî same-storage mirror at a different address (clause 10.5). |
| [`perl_demo.rdl`](perl_demo.rdl) | Perl preprocessor (clause 16.3) generating 8 DMA channels via `<% for my $i ... %>`. Requires `perl` on PATH. |
| [`stress_1000.rdl`](stress_1000.rdl) | 1000 registers √ó 30 fields ‚Äî performance fixture. Generated by `gen_stress.py`. |
| [`gen_stress.py`](gen_stress.py) | Generator for the stress fixture. |

## What to try in each file

**Hover** any keyword (`bridge`, `ispresent`, `alias`, `encode`), property,
access value, or user-defined type identifier ‚Äî explanations + (for
parametrized types) the parameter list.

**Goto-definition** (`F12` / Ctrl-click) on:

- a top-level type name (`my_counter_t`, `power_state`, `irq_status_t`)
- a path expression (`top.LCR.baud` ‚Äî segments resolve through the elaborated tree)
- a signal reference (`pkt_rx_pulse`, `global_rst`)
- a property after `encode = ‚Ä¶`

**Find references** (`Shift+F12`) on a type ‚Äî every instantiation site.

**Rename** (`F2`) on a type ‚Äî workspace-wide rename, refuses on collision.

**Memory Map viewer** (`Ctrl+Shift+V`):

- Click any register row ‚Üí editor scrolls to its declaration with a flash.
- Click an addrmap or regfile row ‚Üí editor reveals its declaration too.
- The chevron (‚ñæ/‚ñ∏) collapses/expands containers without jumping.
- ‚ó∑ / ‚ö° badges next to field names mark counters and interrupts.
- Bit grid wraps at 16 bits per row for 32-/64-bit registers.
- Detail panel: paste a hex value into the binary decoder for live decode.
- Filter (Cmd-F) by name / address / field.

**Code action** (lightbulb) on a field instantiation missing `= 0` ‚Äî quick-fix inserts it.

**Document formatting** (`Shift+Alt+F`) ‚Äî trims trailing whitespace,
normalises tabs, ensures single trailing newline. Conservative.

**Show effective include paths** (Command Palette) ‚Äî quick-pick of the
deduped `include` search path list with provenance (`setting` /
`peakrdl.toml` / `sibling`).
